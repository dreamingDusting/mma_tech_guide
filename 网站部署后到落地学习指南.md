# 网站部署后到落地学习指南

## 前言

经过前面章节的学习，我们已经掌握了开发环境搭建、项目部署的基本技能。但这还差万里长征的最后一步——把网站真正放到互联网上，让所有人都能访问，还需要经历域名购买、备案、解析、CDN配置等一系列流程。

这篇文章，我们将详细介绍网站从开发完成到上线运营的完整流程，包括域名购买、ICP备案、子域名创建、域名解析、公安备案以及CDN配置等内容。快来一起探索吧\~ ✨

## 第一章：域名购买与ICP备案

### 1.1 什么是域名？

域名（Domain Name）就是网站的地址啦\~ 比如 `www.mmaupc.cn`。虽然服务器用的是IP地址（如 `123.45.67.89`），但域名更好记呀，总不能让大家记住一串数字吧？那也太痛苦了qwq

### 1.2 域名购买

在国内买域名，需要找有工信部认证的靠谱服务商哦。常见的有：

- **阿里云万网**：<https://wanwang.aliyun.com>
- **腾讯云**：<https://cloud.tencent.com>
- **华为云**：<https://www.huaweicloud.com>

我们协会的域名就是在阿里云买的！具体步骤超简单：

1. 登录域名服务商网站
2. 搜索想要的域名，看看有没有被抢走
3. 选好年限然后支付\~ money money！
4. 实名认证走起（国内域名必须滴！）

### 1.3 为什么要备案？

没错！在中国大陆运行的网站，是必须备案的！这可是国家规定哦\~ 备案主要是为了规范互联网信息服务，保护网络安全。

**要是不备案会怎样呢？**

- 网站直接被ban掉，打不开
- 搜索引擎可能不收录你，流量就没啦
- 严重的话可能会有行政处罚哦......

所以一定要重视起来！

### 1.4 ICP备案流程

#### 备案类型

备案分两种类型：

- **个人备案**：适合个人博客、个人作品展示这种非营利性的网站
- **企业备案**：公司官网、电商平台这种商业网站用

我们协会的网站是**个人备案**哦\~

#### 备案信息

当当\~ 这是我们的备案信息：

- **备案主体**：个人
- **网站名称**：小数的个人随笔
- **网站备注**：这是一个个人网站，主要用于分享作者本人的数学学习心得、趣味数学随笔和解题思路分享。

#### 备案流程（以阿里云为例）

1. **准备工作**
   - 注册阿里云账号
   - 进行个人实名认证
   - 准备身份证正反面照片\~
2. **提交备案申请**
   - 登录阿里云备案系统
   - 填写网站信息（网站名称、域名、服务器IP等）
   - 上传个人证件材料
   - 填写网站备注说明\~
3. **初审**
   - 阿里云客服会打电话来核实信息哦
   - 记得保持手机畅通，如实回答就好啦\~
4. **管局审核**
   - 初审通过后，进入管局大大审核
   - 审核时间：各省不同，一般7-20个工作日
   - 耐心等待，就会收到备案号啦！🎉
5. **备案成功**
   - 在网站底部添加备案号和链接
   - 链接指向：<http://www.beian.miit.gov.cn>

### 1.5 备案查询

备案完成后，可以在这里查询：

- **工信部备案查询**：<https://beian.miit.gov.cn>
- 各云服务商备案控制台

## 第二章：子域名创建与解析

### 2.1 什么是子域名？

子域名就是主域名的小弟！比如：

- 主域名：`mmaupc.cn`
- 子域名：`www.mmaupc.cn`、`sci.mmaupc.cn`、`cms.mmaupc.cn`

### 2.2 我们的子域名规划

协会网站的子域名规划如下：

| 子域名                                   | 用途        |
| ------------------------------------- | --------- |
| [www.mmaupc.cn](http://www.mmaupc.cn) | 官网主站\~    |
| sci.mmaupc.cn                         | 官网子域名\~   |
| cms.mmaupc.cn                         | 后台管理系统子域名 |

### 2.3 在域名服务商处添加子域名

以阿里云为例，操作步骤：

1. 登录阿里云控制台
2. 进入"域名"服务
3. 点击要管理的域名
4. 进入"DNS设置" → "添加记录"
5. 添加A记录：
   | 记录类型 | 主机记录 | 记录值（服务器IP） |
   | ---- | ---- | ---------- |
   | A    | www  | 你的服务器公网IP  |
   | A    | sci  | 你的服务器公网IP  |
   | A    | cms  | 你的服务器公网IP  |
   | A    | @    | 你的服务器公网IP  |

### 2.4 域名解析

域名解析就是让域名指向服务器IP地址的过程\~ 当用户访问 `www.mmaupc.cn` 时，DNS服务器就会把它翻译成对应的IP地址，这样就能找到我们的服务器啦！

**解析生效时间**：一般10分钟到2小时不等，DNS刷新也是需要时间滴\~

**怎么知道解析有没有生效呢？**

```bash
# Windows
ping www.mmaupc.cn

# Linux/Mac
nslookup www.mmaupc.cn
```

能看到IP就说明成功啦！✨

## 第三章：公安备案

### 3.1 什么是公安备案？

公安备案是依据《计算机信息网络国际联网安全保护管理办法》进行的备案，由公安机关大大们负责管理哦\~ 跟ICP备案不太一样，这个是在当地公安机关办理的。

### 3.2 哪些网站需要公安备案？

根据规定，只要是在中国境内提供互联网信息服务的网站，都需要进行公安备案！简单来说，只要你的网站让人能访问，就要备案！

### 3.3 公安备案流程

1. **注册账号**
   - 访问：<https://www.beian.gov.cn>
   - 注册账号并完成实名认证\~
2. **提交备案信息**
   - 选择备案主体类型（个人/单位）
   - 填写网站信息
   - 填写服务器所在地\~
3. **材料上传**
   - 个人：身份证照片\~
   - 网站安全承诺书（在线签署就好啦）
4. **审核**
   - 审核时间：1-3个工作日，还挺快的！
   - 审核通过后，在网站底部添加公安备案号\~

### 3.4 公安备案号格式

公安备案号是这样子的：鲁公网安备 \[xxxxxxxx] 号（每个省开头不一样啦\~）

## 第四章：CDN配置

### 4.1 什么是CDN？

CDN（Content Delivery Network，内容分发网络）是一个非常厉害的东西！它通过在全球各地部署很多小节点（边缘服务器），把网站内容缓存到离用户最近的地方\~

使用CDN的好处：

- 🚀 **加速访问**：用户从最近的服务器拿资源，速度嗖嗖的！
- 💪 **减轻源站压力**：大量请求CDN节点帮忙扛着，服务器压力小多啦\~
- 🛡️ **隐藏源站IP**：真实服务器IP被藏起来，更安全咯\~
- ⭐ **提升稳定性**：就算某个节点挂了，CDN会自己切换到其他节点\~

### 4.2 CDN工作原理

```
用户访问 www.mmaupc.cn
        ↓
本地DNS解析到CDN节点
        ↓
CDN节点检查缓存
        ↓
[缓存命中] → 直接返回内容~超快！
[缓存未命中] → 回源站获取 → 缓存后返回给用户
```

### 4.3 常见CDN服务商

- **阿里云CDN**：<https://cdn.aliyun.com>
- **腾讯云CDN**：<https://cloud.tencent.com/product/cdn>
- **百度云CDN**：<https://cloud.baidu.com/product/cdn>
- **CloudFlare**：<https://www.cloudflare.com>（国外的，适合外贸站\~）

我们协会用的是阿里云CDN，稳定又靠谱！

### 4.4 CDN配置步骤

下面以阿里云CDN为例，手把手教你配置\~

#### 4.4.1 开通CDN服务

1. 登录阿里云控制台
2. 搜索"CDN"或进入"阿里云CDN"产品页
3. 点击"立即开通"
4. 选择计费方式（按流量/按带宽峰值）
5. 确认开通\~

#### 4.4.2 添加加速域名

每个子域名都要添加为加速域名：

| 子域名                                   | 加速域名                                  | 业务类型 |
| ------------------------------------- | ------------------------------------- | ---- |
| [www.mmaupc.cn](http://www.mmaupc.cn) | [www.mmaupc.cn](http://www.mmaupc.cn) | 网站加速 |
| sci.mmaupc.cn                         | sci.mmaupc.cn                         | 网站加速 |
| cms.mmaupc.cn                         | cms.mmaupc.cn                         | 网站加速 |

**操作步骤**：

1. 在CDN控制台点击"添加域名"
2. 输入要加速的子域名（比如 `www.mmaupc.cn`）
3. 填写基本配置信息\~
4. 选择源站类型（IP/域名/OSS）
5. 填写源站信息（你的服务器IP）
6. 提交审核\~

#### 4.4.3 配置源站信息

**源站信息配置**：

- **源站类型**：选择"IP"
- **源站地址**：填写你的云服务器公网IP\~
- **端口**：80（HTTP）/443（HTTPS）/等等（根据具体情况）

#### 4.4.4 配置缓存规则

合理的缓存配置能让访问速度飞起！✨

| 缓存内容            | 缓存时间 | 说明        |
| --------------- | ---- | --------- |
| 静态资源（图片/CSS/JS） | 1个月  | 很少变化的资源   |
| HTML页面          | 1小时  | 可能频繁更新    |
| API接口           | 不缓存  | 每次都需要最新数据 |

**推荐缓存配置**：

```
# 静态资源
.*\.(jpg|jpeg|png|gif|ico|css|js|svg|woff|woff2|ttf|eot)$
Cache: 30天

# HTML文件
.*\.html$
Cache: 1小时

# 其他
/*
Cache: 1小时
```

#### 4.4.5 配置访问区域

根据目标用户群体来选择\~

| 选项          | 说明         | 适用场景   |
| ----------- | ---------- | ------ |
| 中国大陆        | 只在中国大陆节点加速 | 国内用户多  |
| 全球（不包含中国大陆） | 只在海外节点加速   | 主要面向海外 |
| 全球          | 全球节点都能加速   | 用户分布全球 |

**协会网站建议**：选择"中国大陆"，因为主要访问者是国内用户\~ 考虑到有的同学可能经常挂着vpn也可以选"全球"啦！

#### 4.4.6 获取CNAME

配置完成后，CDN会给你分配一个CNAME地址，类似酱紫：

```
www.mmaupc.cn.abc.def.cdn.cn
```

这个CNAME在后面域名解析的时候要用到哦\~ 记下来！

### 4.5 各子域名CDN配置汇总

为方便管理，这里是三个子域名的配置汇总（示例）：

| 子域名                                   | 加速域名                                  | 源站IP    | CNAME                                                               |
| ------------------------------------- | ------------------------------------- | ------- | ------------------------------------------------------------------- |
| [www.mmaupc.cn](http://www.mmaupc.cn) | [www.mmaupc.cn](http://www.mmaupc.cn) | 你的服务器IP | [www.mmaupc.cn.abc.def.cdn.cn](http://www.mmaupc.cn.abc.def.cdn.cn) |
| sci.mmaupc.cn                         | sci.mmaupc.cn                         | 你的服务器IP | sci.mmaupc.cn.abc.def.cdn.cn                                        |
| cms.mmaupc.cn                         | cms.mmaupc.cn                         | 你的服务器IP | cms.mmaupc.cn.abc.def.cdn.cn                                        |

### 4.6 验证CDN是否生效

配置完等大概10分钟后，可以这样验证CDN有没有生效：

```bash
# 查看域名解析结果
nslookup www.mmaupc.cn

# 如果返回的是CDN节点的IP，而不是源站IP，那就说明CDN已经在工作啦！🎉
```

## 第五章：使用CNAME进行域名解析

### 5.1 什么是CNAME？

CNAME（Canonical Name）记录是DNS家族的重要成员！它能让一个域名指向另一个域名\~

当我们使用CDN时，就不能把域名直接解析到服务器IP啦，要解析到CDN给的CNAME地址，这样用户才能享受到CDN的加速服务\~

### 5.2 配置步骤

1. 登录阿里云域名控制台
2. 进入域名解析设置
3. 把原来的A记录删掉（或者改掉），换成CNAME记录：

| 记录类型  | 主机记录 | 记录值                                                                 |
| ----- | ---- | ------------------------------------------------------------------- |
| CNAME | www  | [www.mmaupc.cn.abc.def.cdn.cn](http://www.mmaupc.cn.abc.def.cdn.cn) |
| CNAME | sci  | sci.mmaupc.cn.abc.def.cdn.cn                                        |
| CNAME | cms  | cms.mmaupc.cn.abc.def.cdn.cn                                        |
| CNAME | @    | mmaupc.cn.abc.def.cdn.cn                                            |

### 5.3 注意事项

- **删除A记录**：添加CNAME记录前，建议先把原有的A记录删掉，不然可能会冲突哦\~
- **两个都要配置**：如果想让 `mmaupc.cn` 和 `www.mmaupc.cn` 都能访问，两个都要配置CNAME记录\~
- **生效时间**：CNAME解析生效通常需要10分钟到2小时，稍微等一等就好啦\~

### 5.4 验证最终效果

所有配置都完成后，验证一下整个流程：

```bash
# 1. 检查域名解析
nslookup www.mmaupc.cn
# 应该返回CDN节点的IP~

# 2. 测试访问
curl -I http://www.mmaupc.cn

# 3. 检查响应头
# 应该能看到类似 "X-Cache: HIT" 的CDN标识，说明CDN正在工作！
```

🎊 恭喜！网站现在已经可以通过域名访问啦！

## 附录：常见问题

### Q1：ICP备案和公安备案有什么区别？

| 对比项   | ICP备案             | 公安备案                  |
| ----- | ----------------- | --------------------- |
| 主管部门  | 工业和信息化部           | 公安机关                  |
| 法律依据  | 《互联网信息服务管理办法》     | 《计算机信息网络国际联网安全保护管理办法》 |
| 备案入口  | beian.miit.gov.cn | beian.gov.cn          |
| 备案号格式 | 京ICP证xxxxxx号      | 京公网安备xxxxxxx号         |

### Q2：备案期间网站可以访问吗？

不行哦\~ ICP备案期间，网站需要关闭。管局审核期间（通常7-20天），网站是没法访问的。备案成功后才能重新开启！

### Q3：使用CDN还需要备案吗？

要的！CDN只是加速服务，不影响备案要求。在中国大陆使用CDN加速，一样需要进行ICP备案哦\~

### Q4：CDN配置错误导致网站打不开怎么办？

别慌别慌！急救措施：

1. 临时把CNAME记录改回A记录，直接解析到源站IP
2. 检查CDN配置哪里出了问题
3. 确认源站是否能正常访问
4. 修复后改回CNAME记录\~

### Q5：A记录和CNAME记录有什么区别？

- **A记录**：直接把域名解析到IP地址
- **CNAME**：把域名解析到另一个域名

用CDN的话必须用CNAME记录，因为CDN的IP地址可能会变来变去的\~
