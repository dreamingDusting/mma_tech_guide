# 数据库学习指南（MySQL + Redis）

## 前言

数据库是现代应用的核心！不管是网站、APP还是后台系统，都离不开数据库。简单来说，数据库就是"电子化的文件柜"，用来存储和管理数据。

这一章我们要学两个数据库：
- **MySQL**：关系型数据库，用来存储业务数据（用户信息、订单、文章等）
- **Redis**：内存数据库，用来缓存热点数据、提升性能

协会的后端项目用的就是MySQL + Redis的组合！学完这篇，你就能明白数据是怎么存储和访问的了。

## 第一章：MySQL基础

### 1.1 什么是MySQL？

MySQL是世界上最流行的开源关系型数据库。特点：
- 开源免费
- 性能优秀
- 生态丰富（各种工具、教程）
- 支持SQL语言

很多知名网站都用MySQL：Facebook、Twitter、YouTube、阿里巴巴...

### 1.3 MySQL基本概念

**数据库（Database）**
- 就像一个文件夹
- 一个项目通常有一个或多个数据库

**表（Table）**
- 就像Excel里的表格
- 有行（记录）和列（字段）

**字段（Column）**
- 表的列
- 定义数据的类型和约束

**记录（Row）**
- 表的行
- 一条具体的数据

**主键（Primary Key）**
- 唯一标识每条记录
- 不能重复，不能为空

## 第二章：SQL基础

### 2.1 创建数据库和表

```sql
-- 创建数据库
CREATE DATABASE mma DEFAULT CHARSET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 使用数据库
USE mma;

-- 创建用户表
CREATE TABLE users (
    uid BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password VARCHAR(100) NOT NULL COMMENT '密码',
    nickname VARCHAR(50) COMMENT '昵称',
    avatar VARCHAR(200) COMMENT '头像URL',
    role VARCHAR(20) DEFAULT 'user' COMMENT '角色',
    qq BIGINT COMMENT 'QQ号',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

### 2.2 插入数据

```sql
-- 插入一条数据
INSERT INTO users (username, password, nickname, role) 
VALUES ('admin', '123456', '管理员', 'admin');

-- 插入多条数据
INSERT INTO users (username, password, nickname) VALUES
('user1', 'pass1', '用户1'),
('user2', 'pass2', '用户2'),
('user3', 'pass3', '用户3');

-- 插入查询结果
INSERT INTO users (username, nickname)
SELECT username, nickname FROM other_table;
```

### 2.3 查询数据

```sql
-- 查询所有数据
SELECT * FROM users;

-- 查询特定列
SELECT username, nickname FROM users;

-- 条件查询
SELECT * FROM users WHERE role = 'admin';
SELECT * FROM users WHERE age >= 18 AND age <= 30;

-- 模糊查询
SELECT * FROM users WHERE nickname LIKE '%张%';

-- 排序
SELECT * FROM users ORDER BY created_at DESC;  -- 降序
SELECT * FROM users ORDER BY created_at ASC;   -- 升序

-- 分页查询
SELECT * FROM users LIMIT 10 OFFSET 0;
-- 或者简写
SELECT * FROM users LIMIT 0, 10;

-- 去重
SELECT DISTINCT role FROM users;

-- 统计数量
SELECT COUNT(*) FROM users;

-- 分组统计
SELECT role, COUNT(*) as count FROM users GROUP BY role;

-- 聚合函数
SELECT 
    COUNT(*) as total,
    AVG(age) as avg_age,
    MAX(age) as max_age,
    MIN(age) as min_age
FROM users;
```

### 2.4 更新数据

```sql
-- 更新单条
UPDATE users SET nickname = '新昵称' WHERE uid = 1;

-- 更新多条
UPDATE users SET role = 'user' WHERE role = 'guest';

-- 同时更新多个字段
UPDATE users 
SET nickname = '管理员', qq = 123456 
WHERE uid = 1;
```

### 2.5 删除数据

```sql
-- 删除数据
DELETE FROM users WHERE uid = 1;

-- 删除所有数据（慎用！）
DELETE FROM users;

-- 清空表（自增ID会重置）
TRUNCATE TABLE users;
```

## 第三章：表关联

### 3.1 为什么要关联？

实际项目中，表和表之间往往有关系：

- 一个用户可以发多篇文章 → 一对多
- 一个文章只有一个作者 → 多对一
- 一个学生可以选多门课，一门课可以被多个学生选 → 多对多

### 3.2 外键约束

```sql
-- 创建文章表（关联用户表）
CREATE TABLE posts (
    pid BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    title VARCHAR(200) NOT NULL,
    content TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(uid) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

`ON DELETE CASCADE`表示删除用户时，该用户的所有文章也会被删除。

### 3.3 JOIN查询

**内连接（INNER JOIN）：**
```sql
-- 查询所有文章，显示作者名字
SELECT p.*, u.username as author_name, u.nickname as author_nickname
FROM posts p
INNER JOIN users u ON p.user_id = u.uid;
```

**左连接（LEFT JOIN）：**
```sql
-- 查询所有用户及其文章（没有文章的用户也显示）
SELECT u.*, p.title as post_title
FROM users u
LEFT JOIN posts p ON u.uid = p.user_id;
```

**右连接（RIGHT JOIN）：**
```sql
-- 查询所有文章及其作者（没有作者的文章也显示）
SELECT p.*, u.username as author_name
FROM posts p
RIGHT JOIN users u ON p.user_id = u.uid;
```

## 第四章：索引和优化

### 4.1 什么是索引？

索引就像书的目录，可以加快查询速度。没有索引的话，MySQL要一条一条地查找（全表扫描）；有了索引，直接定位，速度快很多。

### 4.2 创建索引

```sql
-- 普通索引
CREATE INDEX idx_username ON users(username);

-- 唯一索引
CREATE UNIQUE INDEX idx_username_unique ON users(username);

-- 复合索引（多列）
CREATE INDEX idx_role_created ON users(role, created_at);

-- 删除索引
DROP INDEX idx_username ON users;
```

### 4.3 查看查询计划

```sql
EXPLAIN SELECT * FROM users WHERE username = 'admin';
```

看看有没有用上索引。

### 4.4 SQL优化小技巧

1. **尽量用主键查询**
```sql
-- 快（用主键索引）
SELECT * FROM users WHERE uid = 1;

-- 慢（可能全表扫描）
SELECT * FROM users WHERE nickname = '管理员';
```

2. **避免SELECT ***
```sql
-- 只查询需要的列
SELECT username, nickname FROM users WHERE uid = 1;
```

3. **合理使用LIMIT**
```sql
-- 分页查询
SELECT * FROM users ORDER BY created_at DESC LIMIT 10 OFFSET 0;
```

4. **避免在索引列上使用函数**
```sql
-- 索引会失效
SELECT * FROM users WHERE YEAR(created_at) = 2024;

-- 改用范围查询
SELECT * FROM users WHERE created_at >= '2024-01-01' AND created_at < '2025-01-01';
```

## 第五章：事务

### 5.1 什么是事务？

事务保证一系列操作要么全部成功，要么全部失败。典型例子：转账。

```sql
-- 转账：A账户转给B账户100元
START TRANSACTION;

UPDATE accounts SET balance = balance - 100 WHERE user_id = 1;
UPDATE accounts SET balance = balance + 100 WHERE user_id = 2;

-- 如果都成功，提交
COMMIT;

-- 如果出错，回滚
-- ROLLBACK;
```

### 4个特性（ACID）：
- **A**tomicity（原子性）：要么全成功，要么全失败
- **C**onsistency（一致性）：事务前后数据保持一致
- **I**solation（隔离性）：事务之间互不影响
- **D**urability（持久性）：提交后数据永久保存

## 第六章：Redis基础

### 6.1 什么是Redis？

Redis是一个开源的**内存数据库**，数据存储在内存中，所以读写速度极快！常用于：
- 缓存热点数据
- 计数器
- 分布式锁
- 消息队列
- Session存储

协会用Redis来做缓存，提升访问速度。

### 6.2 安装Redis

**Linux（Ubuntu）：**
```bash
sudo apt update
sudo apt install redis-server

# 启动
sudo systemctl start redis
sudo systemctl enable redis

# 连接
redis-cli
```

**Docker：**
```bash
docker run -d --name mma-redis -p 6379:6379 redis:7-alpine
```

### 6.3 Redis数据类型

**String（字符串）：**
```bash
# 设置值
SET name "张三"

# 获取值
GET name

# 自增/自减
SET count 10
INCR count      # 11
DECR count      # 10
INCRBY count 5  # 15

# 设置过期时间（秒）
SET key "value" EX 60   # 60秒后过期

# 批量操作
MSET key1 "value1" key2 "value2"
MGET key1 key2
```

**Hash（哈希）：**
```bash
# 相当于对象
HSET user:1 name "张三" age "20"

# 获取
HGET user:1 name
HGETALL user:1

# 自增
HINCRBY user:1 age 1
```

**List（列表）：**
```bash
# 插入
LPUSH todos "学习Vue"
LPUSH todos "学习Java"

# 获取
LRANGE todos 0 -1  # 获取所有

# 弹出
LPOP todos
RPOP todos
```

**Set（集合）：**
```bash
# 添加
SADD tags "vue" "react" "angular"

# 获取所有
SMEMBERS tags

# 判断是否存在
SISMEMBER tags "vue"

# 交集、并集、差集
SINTER set1 set2
SUNION set1 set2
SDIFF set1 set2
```

**ZSet（有序集合）：**
```bash
# 添加（带分数）
ZADD leaderboard 100 "用户A"
ZADD leaderboard 90 "用户B"
ZADD leaderboard 80 "用户C"

# 按分数排序获取
ZRANGE leaderboard 0 -1 WITHSCORES

# 获取排名
ZRANK leaderboard "用户A"
```

### 6.4 Redis常用命令

```bash
# 查看所有键

# 查看键类型
TYPE key

# 删除键
DEL key

# 检查键是否存在
EXISTS key

# 设置过期时间
EXPIRE key 60

# 查看剩余时间
TTL key
```

### 7.3 使用RedisTemplate

```java
@Service
public class RedisService {

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    // String操作
    public void setString(String key, String value) {
        redisTemplate.opsForValue().set(key, value);
    }

    public String getString(String key) {
        return (String) redisTemplate.opsForValue().get(key);
    }

    // 设置带过期时间的值
    public void setWithExpire(String key, String value, long timeout, TimeUnit unit) {
        redisTemplate.opsForValue().set(key, value, timeout, unit);
    }

    // Hash操作
    public void setHash(String key, String hashKey, String value) {
        redisTemplate.opsForHash().put(key, hashKey, value);
    }

    public Object getHash(String key, String hashKey) {
        return redisTemplate.opsForHash().get(key, hashKey);
    }

    // 计数器
    public Long increment(String key) {
        return redisTemplate.opsForValue().increment(key);
    }

    // 删除
    public Boolean delete(String key) {
        return redisTemplate.delete(key);
    }
}
```

### 7.4 实际应用场景

**1. 缓存用户信息：**
```java
public User getUserById(Long uid) {
    String cacheKey = "user:" + uid;
    
    // 先从Redis获取
    Object cached = redisTemplate.opsForValue().get(cacheKey);
    if (cached != null) {
        return (User) cached;
    }
    
    // Redis没有，从数据库查询
    User user = userRepository.findById(uid).orElse(null);
    
    // 存入Redis，1小时过期
    if (user != null) {
        redisTemplate.opsForValue().set(cacheKey, user, 1, TimeUnit.HOURS);
    }
    
    return user;
}
```

**2. 验证码/Token：**
```java
public void saveVerificationCode(String phone, String code) {
    // 5分钟过期
    redisTemplate.opsForValue().set("code:" + phone, code, 5, TimeUnit.MINUTES);
}

public boolean verifyCode(String phone, String code) {
    String savedCode = (String) redisTemplate.opsForValue().get("code:" + phone);
    return code.equals(savedCode);
}
```

**3. 接口限流：**
```java
public boolean tryAcquire(String key, int limit, long time, TimeUnit unit) {
    Long count = redisTemplate.opsForValue().increment("rate:" + key);
    if (count == 1) {
        redisTemplate.expire(key, time, unit);
    }
    return count <= limit;
}
```

## 第八章：MySQL + Redis配合使用

### 8.1 典型架构

```
请求 → Redis（缓存） → MySQL（数据库）
      ↓有              ↓没有
      返回              查询MySQL，存入Redis，返回
```

### 8.2 缓存策略

**1. Cache-Aside（旁路缓存）：**
```java
// 读
public User getUser(Long uid) {
    // 1. 先查Redis
    User user = redisTemplate.get("user:" + uid);
    if (user != null) {
        return user;
    }
    
    // 2. Redis没有，查数据库
    user = database.getUser(uid);
    
    // 3. 存入Redis
    if (user != null) {
        redisTemplate.set("user:" + uid, user, 1, TimeUnit.HOURS);
    }
    
    return user;
}

// 写
public void updateUser(User user) {
    // 1. 先更新数据库
    database.updateUser(user);
    
    // 2. 删除缓存（不是更新！）
    redisTemplate.delete("user:" + user.getUid());
}
```

**为什么要删除缓存而不是更新？**
- 如果先更新缓存，第二个请求可能读到旧数据
- 删除缓存后，下次查询会重新加载最新数据

**2. 缓存过期策略：**
- 设置TTL（Time To Live）：如1小时
- 主动过期：数据变更时主动删除
- 定期过期：Redis定期清理
