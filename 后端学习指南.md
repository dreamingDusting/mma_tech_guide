# 后端开发学习指南

## 前言

欢迎来到后端开发的世界！可能你之前听到"后端"这个词会觉得很高大上，其实没那么玄乎。简单来说，前端是用户能看到的部分，后端就是背后处理数据、提供服务的部分。

不过这里还要细说一下，其实很多简单网页是没有后端的，它们叫做静态网页，像网上一些简单的个人博客，其实很多事静态的网页，优点是负载低也方便部署，相反就是所谓的动态网页，网页的内容不是死的，而是可以动态变化的，这份“动态”，就来自于后端等服务

举个例子：你打开协会官网，点击登录，输入用户名密码后点击确认——这个登录界面是前端做的。但是，你的密码对不对、你有没有权限访问，这些判断都是后端在做。后端还要负责把数据存到数据库里，下次登录的时候再把数据拿出来核对。

咱们协会的后端项目用的是**Java语言**加上**Spring Boot框架**，数据库用的是**MySQL**。这一章，我会从零开始带你学完这些知识。

别担心学不会，我会用最通俗的大白话来讲，结合咱们协会的实际代码，保证你能听懂！

## 第一章：Java基础

### 1.1 什么是Java？

Java是一门"面向对象"的编程语言，特点是"一次编译，到处运行"。啥意思呢？你写好的Java代码可以放在Windows、Linux、Mac各种系统上跑，不需要为每个系统单独写代码。这也就是为什么很多企业的后台系统都用Java。

Java的代码文件后缀是`.java`，写完之后需要编译成`.class`文件才能运行。不过我们在开发的时候用IDE（集成开发环境），比如IntelliJ IDEA，点点按钮就能自动编译运行，很方便。

### 1.2 第一个Java程序

让我们来写第一个Java程序！创建个新文件叫`HelloWorld.java`：

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello World!");
    }
}
```

在命令行运行：

```bash
javac HelloWorld.java  # 编译
java HelloWorld         # 运行
```

输出：

```
Hello World!
```

恭喜你，完成了第一个Java程序！虽然只是打印了一行字，但这代表你正式踏入Java世界了。

### 1.8 面向对象——类与对象

这是Java最核心的概念！Java是"面向对象"的语言，一切皆对象。

**类**：是一个模板，定义了对象有什么属性和方法
**对象**：是类的具体实例

```java
// 定义一个"学生"类
public class Student {
    // 属性（特征）
    private String name;    // 姓名
    private int age;        // 年龄
    private String studentId; // 学号
    
    // 构造方法（创建对象时调用）
    public Student(String name, int age, String studentId) {
        this.name = name;
        this.age = age;
        this.studentId = studentId;
    }
    
    // Getter和Setter方法
    public String getName() {
        return name;
    }
    
    public void setName(String name) {
        this.name = name;
    }
    
    public int getAge() {
        return age;
    }
    
    public void setAge(int age) {
        if (age > 0) {
            this.age = age;
        }
    }
    
    // 方法（行为）
    public void study() {
        System.out.println(name + "正在学习");
    }
    
    public void introduce() {
        System.out.println("我叫" + name + "，今年" + age + "岁，学号" + studentId);
    }
}
```

**使用这个类：**

```java
public class Main {
    public static void main(String[] args) {
        // 创建对象
        Student student = new Student("张三", 20, "2021001");
        
        // 调用方法
        student.introduce();
        student.study();
        
        // 修改属性
        student.setName("李四");
        System.out.println(student.getName());
    }
}
```

## 第二章：Spring Boot——让Java Web开发更简单

### 2.1 什么是Spring Boot？

前面学的Java基础是内功心法，现在要学的Spring Boot就是一套"武功套路"。Spring Boot是基于Spring框架的，它帮我们做了很多配置工作，让开发Web应用变得更简单。

以前用纯Spring开发，要配置很多东西：web.xml、applicationContext.xml、数据库连接池等等，光配置就要写半天。Spring Boot的理念是"约定大于配置"，大部分配置都给你默认好了，你只需要关注业务逻辑就行。

### 2.3 Controller——接收请求

Controller负责接收前端发来的请求，然后调用Service处理，最后返回结果。

```java
// 简单的Controller示例
@RestController
@RequestMapping("/api/user")
public class UserController {
    
    @Autowired
    private UserService userService;
    
    // 处理POST请求
    @PostMapping("/create")
    public ResponseEntity<Map<String, Object>> createUser(@RequestBody User user) {
        try {
            User createdUser = userService.createUser(user);
            createdUser = User.getEncryptedUser(createdUser);
            
            Map<String, Object> response = new HashMap<>();
            response.put("success", true);
            response.put("message", "用户创建成功");
            response.put("data", createdUser);
            return ResponseEntity.ok(response);
        } catch (Exception e) {
            Map<String, Object> response = new HashMap<>();
            response.put("success", false);
            response.put("message", e.getMessage());
            return ResponseEntity.badRequest().body(response);
        }
    }
    
    // 处理GET请求
    @GetMapping("/getById")
    public ResponseEntity<Map<String, Object>> getUserById(@RequestParam Long uid) {
        Optional<User> user = userService.getUserById(uid);
        
        Map<String, Object> response = new HashMap<>();
        if (user.isPresent()) {
            response.put("success", true);
            response.put("data", user.get());
        } else {
            response.put("success", false);
            response.put("message", "用户不存在");
        }
        return ResponseEntity.ok(response);
    }
}
```

### 2.4 Service——业务逻辑

Service层负责处理具体的业务逻辑：

```java
@Service
public class UserService {
    
    @Autowired
    private UserRepository userRepository;
    
    // 创建用户
    public User createUser(User user) {
        // 业务逻辑验证
        if (user.getUsername() == null || user.getUsername().isEmpty()) {
            throw new RuntimeException("用户名不能为空");
        }
        
        // 检查用户名是否已存在
        if (userRepository.findByUsername(user.getUsername()).isPresent()) {
            throw new RuntimeException("用户名已存在");
        }
        
        // 保存到数据库
        return userRepository.save(user);
    }
    
    // 获取所有用户
    public List<User> getAllUsers() {
        return userRepository.findAll();
    }
    
    // 根据ID查询
    public Optional<User> getUserById(Long uid) {
        return userRepository.findById(uid);
    }
    
    // 更新用户
    public User updateUser(Long uid, User userDetails) {
        User user = userRepository.findById(uid)
                .orElseThrow(() -> new RuntimeException("用户不存在"));
        
        if (userDetails.getNickname() != null) {
            user.setNickname(userDetails.getNickname());
        }
        if (userDetails.getPassword() != null) {
            user.setPassword(userDetails.getPassword());
        }
        
        return userRepository.save(user);
    }
    
    // 删除用户
    public void deleteUser(Long uid) {
        userRepository.deleteById(uid);
    }
}
```

### 2.5 Repository——数据访问

Repository层负责和数据库打交道。Spring Data JPA让这变得超级简单，只需要定义一个接口就行：

```java
// 继承JpaRepository，自动拥有CRUD方法
public interface UserRepository extends JpaRepository<User, Long> {
    
    // 根据用户名查询（方法名自动解析）
    Optional<User> findByUsername(String username);
    
    // 根据用户名模糊查询
    List<User> findByNicknameContaining(String nickname);
    
    // 根据角色查询
    List<User> findByRole(String role);
    
    // 自定义查询（用JPQL）
    @Query("SELECT u FROM User u WHERE u.username = :username AND u.role = :role")
    Optional<User> findByUsernameAndRole(@Param("username") String username, 
                                          @Param("role") String role);
}
```

### 2.6 Entity——实体类

Entity就是数据库表在Java中的映射，每个Entity对应一张表：

```java
@Entity
@Table(name = "users")
public class User {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long uid;
    
    @Column(nullable = false, unique = true)
    private String username;
    
    @Column(nullable = false)
    private String password;
    
    private String nickname;
    
    private String avatar;
    
    private String role;
    
    private Long qq;
    
    private String securityQuestion;
    
    private String securityAnswer;
    
    // Getter和Setter...
}
```

这里用到了JPA注解：

- `@Entity`：告诉Spring这是实体类
- `@Table(name = "users")`：对应数据库的users表
- `@Id`：主键
- `@GeneratedValue`：主键自增
- `@Column`：列的映射配置

### 3.1 什么是数据库？

数据库就是存储数据的地方。你可以把它想象成一个超级Excel表格，但是功能强大得多：

- 可以存海量数据
- 查询速度快
- 可以多人同时操作
- 数据安全（有备份、有权限控制）

我们用的是MySQL，它是开源的关系型数据库，非常流行。协会还使用了Redis作为缓存来提升性能。

### 3.2 常用操作

**基本SQL语句：**

```sql
-- 增
INSERT INTO users (username, nickname) VALUES ('admin', '管理员');

-- 查
SELECT * FROM users WHERE role = 'admin';

-- 改
UPDATE users SET nickname = '新昵称' WHERE uid = 1;

-- 删
DELETE FROM users WHERE uid = 1;
```

### 3.3 Spring Boot连接数据库

在Spring Boot中，只需要配置一下就能连接数据库：

```yaml
# application.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mma?useSSL=false&serverTimezone=UTC
    username: root
    password: yourpassword
    driver-class-name: com.mysql.cj.jdbc.Driver

  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true
```

然后定义Entity类，Spring Data JPA会自动帮我们操作数据库！

## 第四章：实战——从请求到响应的完整流程

### 4.1 一个完整的请求流程

让我们通过一个具体的例子，看看从前端点击按钮到后端处理再到返回数据的完整流程：

**场景：前端获取所有用户列表**

1. **前端发起请求**（Vue组件中）：

```javascript
// mma_cms/src/utils/api.ts
import axios from 'axios';

const http = axios.create({
    baseURL: 'http://localhost:8080/api',
    timeout: 30000
});

http.interceptors.request.use(config => {
    const token = localStorage.getItem('token');
    if (token) {
        config.headers['Authorization'] = `Bearer ${token}`;
    }
    return config;
});

export default http;

// 调用接口
const response = await http.post('/user/getAll');
```

1. **后端Controller接收请求**：

```java
// UserController.java
@PostMapping("/getAll")
public ResponseEntity<Map<String, Object>> getAllUsers() {
    List<User> users = userService.getAllUsers();
    List<User> encryptedUsers = users.stream()
            .map(User::getEncryptedUser)
            .toList();
    
    Map<String, Object> response = new HashMap<>();
    response.put("success", true);
    response.put("data", encryptedUsers);
    response.put("total", encryptedUsers.size());
    return ResponseEntity.ok(response);
}
```

1. **Service处理业务逻辑**：

```java
// UserService.java
@Service
public class UserService {
    @Autowired
    private UserRepository userRepository;
    
    public List<User> getAllUsers() {
        return userRepository.findAll();
    }
}
```

1. **Repository操作数据库**：

```java
// UserRepository.java
public interface UserRepository extends JpaRepository<User, Long> {
    // findAll()是JpaRepository自带的方法
}
```

1. **数据库返回数据**：

```sql
SELECT * FROM users;
```

1. **数据层层返回**：数据库 → Repository → Service → Controller → 前端
2. **前端处理响应**：

```javascript
const response = await http.post('/user/getAll');
if (response.data.success) {
    this.users = response.data.data;
}
```

### 4.2 实战：添加一个新功能

假设我们现在要在协会官网添加一个"活动报名"功能，需要：

**1. 设计数据库表**：

```sql
CREATE TABLE activities (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    start_time TIMESTAMP,
    end_time TIMESTAMP,
    max_participants INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE activity_registrations (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    activity_id BIGINT,
    user_id BIGINT,
    register_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (activity_id) REFERENCES activities(id),
    FOREIGN KEY (user_id) REFERENCES users(uid)
);
```

**2. 创建Entity**：

```java
@Entity
public class Activity {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String title;
    private String description;
    private LocalDateTime startTime;
    private LocalDateTime endTime;
    private Integer maxParticipants;
    
    // getters and setters
}
```

**3. 创建Repository**：

```java
public interface ActivityRepository extends JpaRepository<Activity, Long> {
}
```

**4. 创建Service**：

```java
@Service
public class ActivityService {
    @Autowired
    private ActivityRepository activityRepository;
    
    public Activity createActivity(Activity activity) {
        return activityRepository.save(activity);
    }
    
    public List<Activity> getAllActivities() {
        return activityRepository.findAll();
    }
    
    public Optional<Activity> getActivityById(Long id) {
        return activityRepository.findById(id);
    }
}
```

**5. 创建Controller**：

```java
@RestController
@RequestMapping("/api/activity")
public class ActivityController {
    
    @Autowired
    private ActivityService activityService;
    
    @PostMapping("/create")
    public ResponseEntity<Map<String, Object>> create(@RequestBody Activity activity) {
        Activity created = activityService.createActivity(activity);
        
        Map<String, Object> response = new HashMap<>();
        response.put("success", true);
        response.put("data", created);
        return ResponseEntity.ok(response);
    }
    
    @GetMapping("/list")
    public ResponseEntity<Map<String, Object>> list() {
        List<Activity> activities = activityService.getAllActivities();
        
        Map<String, Object> response = new HashMap<>();
        response.put("success", true);
        response.put("data", activities);
        return ResponseEntity.ok(response);
    }
}
```

### 4.3 常见问题和解决方案

**1. 前端请求不到后端**

- 检查后端是否启动
- 检查端口是否对应
- 检查跨域配置
- 检查请求路径是否正确

**2. 数据库连接失败**

- 检查MySQL是否启动
- 检查用户名密码
- 检查数据库名称
- 检查URL是否正确

**3. 数据查不到**

- 用`System.out.println()`打印日志
- 查看SQL是否正确
- 检查字段名是否匹配

### 5.5 文件上传

```java
@PostMapping("/upload")
public ResponseEntity<Map<String, Object>> upload(@RequestParam("file") MultipartFile file) {
    try {
        String filename = file.getOriginalFilename();
        String uploadDir = "uploads/";
        File dir = new File(uploadDir);
        if (!dir.exists()) {
            dir.mkdirs();
        }
        
        Path path = Paths.get(uploadDir + filename);
        Files.write(path, file.getBytes());
        
        Map<String, Object> response = new HashMap<>();
        response.put("success", true);
        response.put("filePath", path.toString());
        return ResponseEntity.ok(response);
    } catch (Exception e) {
        Map<String, Object> response = new HashMap<>();
        response.put("success", false);
        response.put("message", e.getMessage());
        return ResponseEntity.status(500).body(response);
    }
}
```
