好！现在你已经学完了前端、后端、数据库、Docker、Nginx各方面的知识。但还有一个关键问题：这些部分怎么"组装"在一起，变成一个完整的系统？

这篇指南就是来解决这个问题的！我们会讲到：

## 第一章：前后端通信基础

### 1.1 什么是API？

API（Application Programming Interface）就是"接口"，说白了就是一套规则——前端怎么请求、后端怎么响应，都有明确规定。

协会的后端API大致长这样：

```
http://localhost:8080/api/user/getAll
http://localhost:8080/api/user/create
http://localhost:8080/api/news/getAll
```

- `http://localhost:8080` - 后端地址
- `/api` - 接口前缀
- `/user` - 用户模块
- `/getAll` - 获取所有

### 1.2 HTTP请求方法

| 方法     | 用途   | 示例      |
| ------ | ---- | ------- |
| GET    | 获取数据 | 查询用户列表  |
| POST   | 创建数据 | 用户登录、注册 |
| PUT    | 更新数据 | 修改用户信息  |
| DELETE | 删除数据 | 删除用户    |

### 1.3 请求和响应格式

**请求（前端发给后端）：**

```json
// POST请求体
{
    "username": "admin",
    "password": "123456"
}
```

**响应（后端返回给前端）：**

```json
{
    "success": true,
    "message": "登录成功",
    "data": {
        "uid": 1,
        "username": "admin",
        "nickname": "管理员"
    }
}
```

## 第二章：跨域问题详解

### 2.1 什么是同源策略？

浏览器的安全机制之一叫"同源策略"。简单说就是：

- 只有**同源**的网页才能互相访问
- 不同源的网页之间不能随意发送请求

**什么是同源？**

- 协议相同（http vs https）
- 域名相同（localhost vs example.com）
- 端口相同（8080 vs 80）

举个例子：

- 前端：<http://localhost:5173>
- 后端：<http://localhost:8080>

这两个**不同源**！因为端口不一样（5173 vs 8080）。直接请求会报CORS错误！

### 2.2 为什么会报跨域错误？

```
前端(5173)  →  浏览器  →  后端(8080)
               ↓
           浏览器：不行！你俩不同源！
           (╯°□°)╯︵ ┻━┻
```

浏览器会阻止这种请求，并报错：

```
Access to fetch at 'http://localhost:8080/api/user/getAll' 
from origin 'http://localhost:5173' has been blocked by CORS policy
```

### 2.3 解决方案

**方案一：后端配置CORS**

在后端添加跨域配置：

```java
// Spring Boot配置CORS
@Configuration
public class CorsConfig implements WebMvcConfigurer {
    
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
                .allowedOrigins("http://localhost:5173", "http://localhost:80")
                .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
                .allowedHeaders("*")
                .allowCredentials(true)
                .maxAge(3600);
    }
}
```

或者用注解（简单）：

```java
@CrossOrigin(origins = "http://localhost:5173")
@RestController
@RequestMapping("/api/user")
public class UserController {
    // ...
}
```

**方案二：Nginx反向代理（推荐）**

让前端请求Nginx，Nginx转发到后端。这样对浏览器来说都是同源！

```nginx
server {
    listen 80;
    server_name localhost;

    # 前端静态文件
    location / {
        root /usr/share/nginx/html;
        index index.html;
    }

    # API请求转发到后端
    location /api/ {
        proxy_pass http://backend:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

前端请求时：

```javascript
// 开发环境（Vite代理）
// vite.config.js
export default {
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true
      }
    }
  }
}
```

然后前端直接请求 `/api/xxx` 就行，Vite会帮你转发到后端。

## 第三章：前端对接后端

### 3.1 项目中的实际用法

来看看协会前端是怎么对接后端的：

**1. 配置axios（http.js）：**

```javascript
// mma_web/src/api/http.js
import axios from 'axios';

const http = axios.create({
    baseURL: 'http://localhost:8080/api',  // 后端地址
    timeout: 30000,
    headers: {
        'Content-Type': 'application/json'
    }
});

// 请求拦截器：自动加上token
http.interceptors.request.use(config => {
    const token = localStorage.getItem('token');
    if (token) {
        config.headers['Authorization'] = `Bearer ${token}`;
    }
    return config;
});

// 响应拦截器：统一处理错误
http.interceptors.response.use(
    response => response.data,
    error => {
        if (error.response?.status === 401) {
            // token过期，跳转登录
            localStorage.removeItem('token');
            window.location.href = '/login';
        }
        return Promise.reject(error);
    }
);

export default http;
```

**2. 调用API：**

```javascript
// 获取用户列表
const response = await http.post('/user/getAll');

// 登录
const response = await http.post('/login', {
    username: 'admin',
    password: '123456'
});

// 上传文件
const formData = new FormData();
formData.append('file', file);
const response = await http.post('/upload', formData, {
    headers: { 'Content-Type': 'multipart/form-data' }
});
```

### 3.2 完整示例：登录功能

**前端（Vue）：**

```vue
<template>
  <div class="login">
    <input v-model="form.username" placeholder="用户名" />
    <input v-model="form.password" type="password" placeholder="密码" />
    <button @click="handleLogin">登录</button>
  </div>
</template>

<script setup>
import { reactive } from 'vue';
import http from '@/api/http';
import { useUserStore } from '@/stores/user';

const form = reactive({
  username: '',
  password: ''
});

const userStore = useUserStore();

const handleLogin = async () => {
  try {
    const response = await http.post('/login', form);
    if (response.success) {
      // 保存登录信息
      userStore.setUser({
        uid: response.data.uid,
        token: response.data.token,
        nickname: response.data.nickname
      });
      // 跳转首页
      router.push('/');
    } else {
      alert(response.message);
    }
  } catch (error) {
    alert('登录失败');
  }
};
</script>
```

**后端（Spring Boot）：**

```java
@PostMapping("/login")
public ResponseEntity<Map<String, Object>> login(@RequestBody Map<String, String> loginData) {
    String username = loginData.get("username");
    String password = loginData.get("password");
    
    // 查询用户
    Optional<User> userOpt = userService.getUserByUsername(username);
    
    if (userOpt.isEmpty()) {
        return ResponseEntity.ok(Map.of("success", false, "message", "用户不存在"));
    }
    
    User user = userOpt.get();
    if (!user.getPassword().equals(password)) {
        return ResponseEntity.ok(Map.of("success", false, "message", "密码错误"));
    }
    
    // 生成token（简化版）
    String token = UUID.randomUUID().toString();
    
    // 返回结果
    Map<String, Object> result = new HashMap<>();
    result.put("success", true);
    result.put("message", "登录成功");
    result.put("data", Map.of(
        "uid", user.getUid(),
        "username", user.getUsername(),
        "nickname", user.getNickname(),
        "token", token
    ));
    
    return ResponseEntity.ok(result);
}
```

## 第四章：后端连接数据库

### 4.1 Spring Boot连接MySQL

**1. 添加依赖（pom.xml）：**

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>

<dependency>
    <groupId>com.mysql</groupId>
    <artifactId>mysql-connector-j</artifactId>
    <scope>runtime</scope>
</dependency>
```

**2. 配置连接信息（application.yml）：**

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mma?useSSL=false&serverTimezone=UTC&characterEncoding=utf8mb4
    username: root
    password: yourpassword
    driver-class-name: com.mysql.cj.jdbc.Driver

  jpa:
    hibernate:
      ddl-auto: update  # 自动建表
    show-sql: true     # 显示SQL
    properties:
      hibernate:
        format_sql: true
        dialect: org.hibernate.dialect.MySQL8Dialect
```

**3. 定义Entity：**

```java
@Entity
@Table(name = "users")
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long uid;
    
    @Column(nullable = false, unique = true)
    private String username;
    
    @Column(nullable = false)
    private String password;
    
    private String nickname;
    private String avatar;
    private String role;
    
    // getter/setter...
}
```

**4. 定义Repository：**

```java
@Repository
public interface UserRepository extends JpaRepository<User, Long> {
    Optional<User> findByUsername(String username);
    boolean existsByUsername(String username);
}
```

然后在Service里直接调用Repository的方法就行！

### 4.2 Spring Boot连接Redis

**1. 添加依赖：**

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>

```

**2. 配置（application.yml）：**

```yaml
spring:
  redis:
    host: localhost
    port: 6379
    password:  # 如果有密码
    database: 0
```

**3. 使用：**

```java
@Service
public class CacheService {
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    public void cacheUser(User user) {
        redisTemplate.opsForValue().set("user:" + user.getUid(), user, 1, TimeUnit.HOURS);
    }
    
    public User getCachedUser(Long uid) {
        return (User) redisTemplate.opsForValue().get("user:" + uid);
    }
}
```

## 第五章：QQ机器人与后端对接

### 5.1 机器人怎么调用后端API？

协会的秘书处插件支持Webhook，可以接收外部系统的请求：

```python
# 机器人Web服务
from aiohttp import web

async def handle_webhook(self, request):
    data = await request.json()
    # 处理外部请求
    return web.json_response({"status": "ok"})
```

### 5.2 后端调用机器人

如果后端需要通知机器人发消息，可以调用机器人的Webhook：

```java
@Service
public class RobotService {
    
    @Value("${robot.webhook.url}")
    private String webhookUrl;
    
    public void sendMessageToGroup(Long groupId, String message) {
        HttpClient client = HttpClient.newHttpClient();
        String json = String.format(
            "{\"group_id\": %d, \"message\": \"%s\"}",
            groupId, message
        );
        
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create(webhookUrl))
            .header("Content-Type", "application/json")
            .POST(HttpRequest.BodyPublishers.ofString(json))
            .build();
        
        client.sendAsync(request, HttpResponse.BodyHandlers.ofString());
    }
}
```

### 5.3 机器人与前端对接

前端可以通过HTTP请求调用机器人的Webhook：

```javascript
// 前端通知机器人
async function notifyRobot(message) {
    await fetch('http://your-robot:8081/webhook', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ message })
    });
}
```

## 第六章：Nginx整合所有服务

### 6.1 架构图

```
用户访问
    ↓
Nginx (80/443端口)
    ├── /  → 前端静态文件
    ├── /api/  → 后端服务 (8080)
    └── /robot/  → 机器人Web服务 (8081)
```

### 6.2 完整Nginx配置

```nginx
# upstream配置（负载均衡）
upstream backend {
    server backend:8080;
}

# 前端静态文件
server {
    listen 80;
    server_name mma.example.com;

    # 前端
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
        
        # 静态资源缓存
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 7d;
            add_header Cache-Control "public, immutable";
        }
    }

    # API反向代理
    location /api/ {
        proxy_pass http://backend/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 文件上传代理
    location /upload/ {
        proxy_pass http://backend/upload/;
        client_max_body_size 100m;
    }

    # 机器人Web服务
    location /robot/ {
        proxy_pass http://robot:8081/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 6.3 SSL/HTTPS配置

```nginx
# HTTP转HTTPS
server {
    listen 80;
    server_name mma.example.com;
    return 301 https://$server_name$request_uri;
}

# HTTPS服务器
server {
    listen 443 ssl http2;

    # SSL证书
    ssl_certificate /etc/nginx/ssl/mma.example.com.crt;

    ssl_prefer_server_ciphers off;

    # HSTS
    add_header Strict-Transport-Security "max-age=31536000" always;
```

## 第七章：Docker Compose一键部署

### 7.1 docker-compose.yaml

这是协会使用的完整部署配置：

```yaml
version: '3.8'

services:
  # MySQL数据库
  mysql:
    image: mysql:8.0
    container_name: mma-mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: mma
      MYSQL_USER: ${MYSQL_USER}
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    command:
      - --character-set-server=utf8mb4
      - --collation-server=utf8mb4_unicode_ci
      - --default-authentication-plugin=mysql_native_password
    networks:
      - mma-network
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

  # Redis缓存
  redis:
    image: redis:7-alpine
    container_name: mma-redis
    restart: always
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    networks:
      - mma-network
    command: redis-server --appendonly yes
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  # 后端服务
  backend:
    build:
      context: ./mma_back_end
      dockerfile: Dockerfile
    container_name: mma-backend
    restart: always
    ports:
      - "8080:8080"
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/mma?useSSL=false&serverTimezone=UTC&characterEncoding=utf8mb4
      SPRING_DATASOURCE_USERNAME: ${MYSQL_USER}
      SPRING_DATASOURCE_PASSWORD: ${MYSQL_PASSWORD}
      SPRING_REDIS_HOST: redis
      SPRING_REDIS_PORT: 6379
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy
    networks:
      - mma-network

  # 前端 + Nginx
  frontend:
    build:
      context: ./mma_web
      dockerfile: Dockerfile
    container_name: mma-frontend
    restart: always
    ports:
      - "80:80"
    depends_on:
      - backend
    networks:
      - mma-network

networks:
  mma-network:
    driver: bridge

volumes:
  mysql-data:
  redis-data:
```

## 第八章：完整数据流示例

### 8.1 用户登录流程

```
1. 用户在浏览器输入 http://localhost
   ↓
2. Nginx接收请求，转发到前端容器
   ↓
3. 前端返回登录页面
   ↓
4. 用户输入用户名密码，点击登录
   ↓
5. 前端发送 POST http://localhost/api/login
   ↓
6. Nginx收到 /api/ 请求，转发到后端:8080
   ↓
7. 后端Controller接收请求
   ↓
8. 后端查询MySQL验证用户名密码
   ↓
9. 后端生成Token，存入Redis
   ↓
10. 后端返回响应给Nginx
   ↓
11. Nginx返回给前端
   ↓
12. 前端保存Token到localStorage，跳转首页
```

### 8.2 获取数据流程

```
1. 前端发起 GET /api/news/getAll
   ↓
2. Nginx转发到后端
   ↓
3. 后端先查Redis缓存
   ↓
   ├── 缓存存在 → 直接返回
   │
   └── 缓存不存在 → 查MySQL → 返回数据 → 存入Redis
   ↓
4. 数据返回给前端渲染
```

### 8.3 机器人通知流程

```
1. 后端需要通知某个群
   ↓
2. 后端调用机器人Webhook API
   ↓
3. 机器人收到请求
   ↓
4. 机器人发送消息到指定群
```

## 附录：常见问题汇总

### 1. 端口被占用

```bash
# 查看端口占用
netstat -tlnp | grep 8080

# 杀死进程
kill -9 <PID>
```

### 2. Docker网络问题

```bash
# 查看网络
docker network ls

# 检查容器网络
docker inspect <container-name> | grep IPAddress
```

### 3. 数据库连接失败

- 检查MySQL是否启动
- 检查用户名密码
- 检查数据库名称
- 检查Docker网络是否连通

### 4. 跨域问题

- 后端添加CORS配置
- 或使用Nginx代理

### 5. 前端请求404

- 检查Nginx配置
- 检查后端是否启动
- 检查API路径

## 结语

到这里，你应该对整个系统的架构和协作方式有了清晰的认识！总结一下：

1. **前端 → 后端**：通过HTTP请求，使用JSON格式通信
2. **跨域问题**：后端配置CORS或用Nginx代理
3. **后端 → 数据库**：Spring Data JPA操作MySQL/Redis
4. **机器人 ↔ 后端**：通过Webhook API调用
5. **部署**：Docker Compose一键启动所有服务

2. 尝试配置Nginx
