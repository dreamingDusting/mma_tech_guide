- AstrBot（QQ机器人）

让我们开始吧！

## 第一章：Docker安装

### 1.1 为什么要安装Docker？

Docker是最重要的工具！它可以：

- 快速部署MySQL、Redis、Nginx等服务
- 保证开发环境和生产环境一致
- 避免"在我电脑上能运行"的问题

### 1.2 Windows安装

**方式一：Docker Desktop（推荐）**

1. 去官网下载：<https://www.docker.com/products/docker-desktop>
2. 运行安装程序
3. 安装完成后启动Docker Desktop
4. 验证：打开终端，输入 `docker --version`

**可能遇到的问题：**

如果启动时报错"WSL2 installation is incomplete"，需要：

1. 开启Windows虚拟化：控制面板 → 程序 → 启用或关闭Windows功能 → 勾选"虚拟机平台"和"适用于Linux的Windows子系统"
2. 安装WSL2：管理员权限打开PowerShell，执行：
   ```powershell
   wsl.exe --install -d Ubuntu
   ```
3. 设置开机启动Hypervisor：
   ```powershell
   bcdedit /set hypervisorlaunchtype auto
   ```
4. 重启电脑

### 1.3 Mac安装

1. 去官网下载：<https://www.docker.com/products/docker-desktop>
2. 安装Docker.dmg
3. 启动Docker Desktop
4. 验证：`docker --version`

### 1.4 Linux安装（Ubuntu）

```bash
# 更新apt
sudo apt update

# 安装必要依赖
sudo apt install apt-transport-https ca-certificates curl software-properties-common

# 添加Docker官方GPG密钥
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# 添加Docker仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装Docker
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 启动Docker
sudo systemctl start docker
sudo systemctl enable docker

# 验证
docker --version
```

### 1.5 Docker镜像加速

国内访问Docker Hub很慢，需要配置镜像加速。在Docker Desktop中：

1. 点击Docker图标 → Settings → Docker Engine
2. 添加以下配置：

```json
{
  "registry-mirrors": [
    "https://registry.docker-cn.com",
    "https://ud6340vz.mirror.aliyuncs.com",
    "https://docker.mirrors.ustc.edu.cn"
  ]
}
```

### 1.6 验证安装

```bash
# 运行测试容器
docker run hello-world

# 查看版本
docker --version
docker-compose --version
```

## 第二章：Vue安装

### 2.1 安装Node.js

Vue需要Node.js环境。Node.js自带npm（包管理器）。

**方式一：使用nvm管理版本（推荐）**

nvm是Node.js版本管理工具，可以让你在同一台电脑上轻松切换不同版本的Node.js，特别适合需要维护多个项目的同学。详见官网：https://www.nvmnode.com/guide/introduction.html

```bash
# 1. 先安装nvm（Windows用户去这里下载：https://github.com/coreybutler/nvm-windows/releases）
# Mac/Linux用户运行：
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 2. 安装Node.js LTS版本
nvm install 18
nvm install 20

# 3. 切换到指定版本
nvm use 20

# 4. 验证
node --version
npm --version
```

> 💡 提示：nvm的好处是可以随时切换版本，不同项目用不同版本的Node.js也不怕冲突！

**方式二：直接安装（如果只是想更快学习）**

如果觉得nvm太麻烦，直接去官网下载也可以：

1. 去官网下载：https://nodejs.org/zh-cn/
2. 推荐安装LTS（长期支持）版本
3. 一路Next点下去就OK

安装完成后验证：

```bash
node --version
npm --version
```

### 2.2 创建Vue项目

**方式一：使用Vite（推荐，更快）**

```bash
# 创建项目
npm create vue@latest

# 或者（指定项目名）
npm create vue@latest my-project

# 进入项目目录
cd my-project

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

**方式二：使用Vue CLI**

```bash
# 全局安装Vue CLI
npm install -g @vue/cli

# 创建项目
vue create my-project

# 进入项目
cd my-project

# 启动
npm run serve
```

### 2.3 Vue官方文档

详细用法请参考：<https://cn.vuejs.org/guide/quick-start.html>

## 第三章：Element Plus安装

### 3.1 安装Element Plus

在Vue项目中安装：

```bash
# 进入Vue项目目录
cd my-project

# 安装Element Plus
npm install element-plus
```

### 3.2 引入Element Plus

**全局引入（main.js）：**

```javascript
import { createApp } from 'vue'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'
import App from './App.vue'

const app = createApp(App)
app.use(ElementPlus)
app.mount('#app')
```

**按需引入（推荐）：**

需要先安装自动导入插件：

```bash
npm install -D unplugin-vue-components unplugin-auto-import
```

然后在vite.config.js中配置：

```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import AutoImport from 'unplugin-auto-import/vite'
import Components from 'unplugin-vue-components/vite'
import { ElementPlusResolver } from 'unplugin-vue-components/resolvers'

export default defineConfig({
  plugins: [
    vue(),
    AutoImport({
      resolvers: [ElementPlusResolver()],
    }),
    Components({
      resolvers: [ElementPlusResolver()],
    }),
  ],
})
```

### 3.4 Element Plus官方文档

详细文档请参考：<https://element-plus.org/zh-CN/guide/quickstart.html>

## 第四章：MySQL安装

### 4.1 Docker方式安装（推荐）

最简单的方式，用Docker：

```bash
# 启动MySQL容器
docker run -d \
  --name mma-mysql \
  -e MYSQL_ROOT_PASSWORD=root123 \
  -e MYSQL_DATABASE=mma \
  -e MYSQL_USER=mma_user \
  -e MYSQL_PASSWORD=mma123 \
  -p 3306:3306 \
  mysql:8.0

# 验证运行
docker ps

# 连接MySQL
docker exec -it mma-mysql mysql -u root -proot123
```

### 4.2 Windows/Mac安装包方式

1. 去官网下载：<https://www.mysql.com/cn/downloads/>
2. 选择"MySQL Community Server"
3. 安装时选择"Full"类型
4. 设置root密码
5. 安装完成后打开MySQL Workbench连接

### 4.3 Linux安装

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install mysql-server

# 启动
sudo systemctl start mysql
sudo systemctl enable mysql

# 安全初始化
sudo mysql_secure_installation

# 登录
sudo mysql -u root -p
```

### 4.4 常用MySQL命令

```bash
# Docker中运行SQL
docker exec -it mma-mysql mysql -u root -proot123 -e "SHOW DATABASES;"

# 导入SQL文件
docker exec -i mma-mysql mysql -u root -proot123 mma < backup.sql

# 导出数据库
docker exec mma-mysql mysqldump -u root -proot123 mma > backup.sql
```

### 4.5 MySQL官方文档

<https://www.mysql.com/cn/>

## 第七章：Spring Boot环境配置

### 7.1 安装JDK

Spring Boot需要JDK 17+（建议JDK 21）。

**Windows/Mac：**

1. 去官网下载JDK：<https://www.oracle.com/java/technologies/downloads/>
2. 或者安装OpenJDK：<https://adoptium.net/>
3. 配置环境变量

**Linux：**

```bash
# Ubuntu
sudo apt install openjdk-21-jdk

# 验证
java --version
```

### 7.2 安装IntelliJ IDEA

推荐使用IDEA作为Java开发工具。

1. 去官网下载：<https://www.jetbrains.com.cn/en-us/idea/>
2. 安装IDEA Ultimate或Community版本
3. 学生可以申请免费教育许可证：<https://www.jetbrains.com/zh-cn/community/education/>

### 7.3 创建Spring Boot项目

**方式一：IDEA创建**

1. 打开IDEA → File → New → Project
2. 选择Spring Initializr
3. 填写项目信息，选择依赖
4. 完成

**方式二：在线创建**

1. 访问：<https://start.spring.io/>
2. 选择项目类型（Maven/Gradle）、语言、版本
3. 添加依赖（Spring Web、Spring Data JPA、MySQL Driver、Redis等）
4. 点击GENERATE下载项目

**方式三：命令行**

```bash
# 安装Spring Boot CLI
curl -s "https://get.sdkman.io" | bash
source "$HOME/.sdkman/bin/sdkman-init.sh"
sdk install springboot

# 创建项目
spring init --name my-project --dependencies web,jpa,mysql,redis my-project
```

### 7.4 IDEA配置

1. 导入项目：File → Open → 选择项目目录
2. 等待Maven/Gradle下载依赖
3. 配置运行：Run → Edit Configurations → + → Spring Boot
4. 运行项目

### 7.5 Spring Boot官方文档

<https://spring.io/projects/spring-boot>

## 第八章：AstrBot安装

### 8.1 Docker方式安装（推荐）

**前置条件：需要先安装Docker（见第一章）**

**安装NapCat（QQ协议端）：**

```bash
# 创建目录
mkdir -p ~/napcat && cd ~/napcat

# 拉取NapCat Docker
docker pull mlikiowa/napcat:latest

# 启动
docker run -d \
  --name napcat \
  -p 6099:6099 \
  -p 6098:6098 \
  -v $(pwd)/config:/app/config \
  -v $(pwd)/data:/app/data \
  --restart unless-stopped \
  mlikiowa/napcat:latest
```

**安装AstrBot：**

```bash
# 创建目录
mkdir -p ~/astrbot && cd ~/astrbot

# 拉取AstrBot
docker pull soulter/astrbot:latest

# 启动AstrBot
docker run -d \
  --name astrbot \
  -p 6185:6185 \
  -p 6199:6199 \
  -v $(pwd)/data:/AstrBot/data \
  -v /etc/localtime:/etc/localtime:ro \
  -v /etc/timezone:/etc/timezone:ro \
  --restart unless-stopped \
  soulter/astrbot:latest
```

**如果网络不好，使用镜像：**

```bash
docker pull m.daocloud.io/docker.io/soulter/astrbot:latest
```

### 8.2 Docker Compose方式（推荐）

同时部署AstrBot和NapCat：

```bash
mkdir astrbot
cd astrbot
wget https://raw.githubusercontent.com/NapNeko/NapCat-Docker/main/compose/astrbot.yml
sudo docker compose -f astrbot.yml up -d
```

### 8.3 访问AstrBot管理面板

1. 启动后等待1-2分钟
2. 访问 <http://localhost:6185>
3. 默认用户名：astrbot
4. 默认密码：astrbot

### 8.4 AstrBot配置

1. 登录管理面板
2. 配置NapCat连接
3. 安装/启用插件
4. 配置需要接入的QQ机器人账号

详细配置请参考：AstrBot官方文档

## 第九章：一键启动所有服务

### 9.1 使用Docker Compose

创建一个docker-compose.yaml文件：

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: mma-mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: mma
      MYSQL_USER: mma_user
      MYSQL_PASSWORD: mma123
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql
    networks:
      - mma-network

  redis:
    image: redis:7-alpine
    container_name: mma-redis
    restart: always
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    networks:
      - mma-network

  nginx:
    image: nginx:latest
    container_name: mma-nginx
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
    depends_on:
      - mysql
      - redis
    networks:
      - mma-network

networks:
  mma-network:
    driver: bridge

volumes:
  mysql-data:
  redis-data:
```

启动所有服务：

```bash
docker-compose up -d
```

### 9.2 验证服务

```bash
# 查看运行中的容器
docker ps

# 测试MySQL
docker exec -it mma-mysql mysql -u root -proot123 -e "SHOW DATABASES;"

# 测试Redis
docker exec -it mma-redis redis-cli PING

# 测试Nginx
curl http://localhost
```

## 第十章：常见问题汇总

### 1. Docker占用C盘空间

```bash
# 清理未使用的镜像
docker image prune -a

# 清理未使用的容器
docker container prune

# 清理构建缓存
docker builder prune
```

### 2. 端口被占用

```bash
# Windows
netstat -ano | findstr 8080
taskkill /PID <PID> /F

# Linux
sudo lsof -i :8080
sudo kill -9 <PID>
```

### 3. Docker权限问题（Linux）

```bash
# 将当前用户加入docker组
sudo usermod -aG docker $USER

# 重新登录
```

### 4. 数据库连接超时

检查：

- MySQL/Redis容器是否启动
- IP地址是否正确（容器间用服务名，不用localhost）
- 防火墙是否阻止

### 5. 跨域问题

后端添加CORS配置，或使用Nginx代理
